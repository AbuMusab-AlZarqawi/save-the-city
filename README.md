# 🏙 Save The City — Onchain AI Game on Ritual Chain

A fully onchain AI game where the city is under threat, you pick a hero, and Ritual Chain's native LLM precompile generates a unique story of how your hero saved the day — permanently recorded on the blockchain.

**Every story is:**
- Generated by AI **inside a Trusted Execution Environment (TEE)**
- Cryptographically tied to your transaction
- Stored permanently onchain
- Costs **0.01 RITUAL** per play

---

## PART 2 — Setting Up the Smart Contract (Hardhat)

### Step 4: Install Contract Dependencies

You are inside the `save-the-city` folder. Run:

```powershell

```

Wait for it to finish. You'll see a lot of text — that's normal.

---

### Step 5: Create Your Contract Environment File

In PowerShell, run:

```powershell
Copy-Item .env.example .env
```

Now open the `.env` file. In VS Code, you can open it by typing:

```powershell
code .env
```

It will look like this:

```
PRIVATE_KEY=your_private_key_here_without_0x_prefix
RITUAL_RPC_URL=https://rpc.ritualfoundation.org
```

**You need to fill in your PRIVATE_KEY.**

To get your private key from MetaMask:
1. Open MetaMask
2. Click the three dots `...` next to your account name
3. Click **Account Details**
4. Click **Export Private Key**
5. Enter your MetaMask password
6. Copy the key (it's a long string of letters and numbers)
7. Paste it after `PRIVATE_KEY=` in the `.env` file

> ⚠️ **IMPORTANT**: Your private key gives full access to your wallet. NEVER share it. NEVER commit it to GitHub. The `.gitignore` file already protects you from accidentally uploading it.

Save the `.env` file.

## How It Works Technically

1. **Player connects wallet** via RainbowKit (MetaMask on Ritual Chain)
2. **Frontend picks a random scenario** from 20 threat scenarios
3. **Player selects a hero** (Dunken, Stefan, or Jez)
4. **Frontend calls `playGame(heroId, scenario)`** with 0.01 RITUAL attached
5. **Contract receives payment**, builds a prompt, and calls Ritual's **native LLM precompile** at address `0x0000000000000000000000000000000000000101`
6. The LLM precompile runs **inside a TEE** — the story is generated and cryptographically attested
7. The story is **returned to the contract** and stored onchain with the player's wallet, hero, scenario, and timestamp
8. The `GamePlayed` event is emitted — frontend reads it from the transaction receipt and displays the story with a typewriter animation

## Troubleshooting

**"Insufficient funds" when deploying:**
→ You need RITUAL tokens in your MetaMask wallet on Ritual Chain Testnet.

**"Cannot connect to network" when deploying:**
→ Check your internet connection. The RPC URL `https://rpc.ritualfoundation.org` must be reachable.

**MetaMask shows wrong network:**
→ In MetaMask, click the network dropdown at the top and select "Ritual Chain Testnet". If it's not there, add it manually with Chain ID 1979.

**Transaction fails with "send exactly 0.01 RITUAL":**
→ The contract requires exactly 0.01 RITUAL. Make sure you have enough balance.

**Story doesn't appear after transaction:**
→ The LLM precompile on testnet may have latency. Wait a moment and check the transaction on the block explorer. If the event has no story field, the precompile may still be warming up on this testnet deployment.

**Vercel build fails:**
→ Make sure the Root Directory is set to `frontend` in Vercel project settings, and all three environment variables are set correctly.

**WalletConnect popup doesn't appear:**
→ Make sure `NEXT_PUBLIC_WALLETCONNECT_PROJECT_ID` is set to a valid project ID from https://cloud.walletconnect.com

## Important Notes on Ritual Chain LLM Precompile

- The precompile address `0x0000000000000000000000000000000000000101` is the **testnet address** — verify this against official Ritual Chain documentation before mainnet deployment.
- The model name `llama3.1-8b-instruct` is the default supported model. Check https://docs.ritual.net for the full list.
- Stories are generated synchronously — the transaction waits for the AI response before confirming. This means transactions take longer than normal (10-60 seconds is normal).
- Each story is unique because the prompt includes the exact scenario text and hero description.

## Security

- Your `.env` and `.env.local` files are in `.gitignore` and will NOT be pushed to GitHub.
- The contract owner (your wallet) is the only address that can call `withdraw()`.
- The LLM precompile runs inside a TEE — outputs are cryptographically attested and cannot be tampered with.
